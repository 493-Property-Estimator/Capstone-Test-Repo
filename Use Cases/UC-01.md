# Use Case UC-01: Enter Street Address to Estimate Property Value

**Goal in Context**: Allow a general user to input a valid street
address so the system can estimate the property value without requiring
geographic coordinates.\
**Scope**: Property Value Estimator System\
**Level**: User Goal\
**Primary Actor**: General User\
**Secondary Actors**: Geocoding Service\
**Trigger**: The user selects the option to estimate a property value
and chooses to enter a street address.

## Success End Condition

-   The system displays an estimated property value (with range) for the
    entered address.
-   The address is successfully geocoded and normalized to a canonical
    location ID.

## Failed End Condition

-   No estimate is produced.
-   The system informs the user that the address could not be validated
    or geocoded.

## Preconditions

-   The Property Value Estimator system is operational.
-   The geocoding service is available.
-   The user has access to the estimate interface.

## Main Success Scenario

1.  The user selects the "Estimate Property Value" option.
2.  The system prompts the user to enter a street address.
3.  The user enters a street address.
4.  The system validates the address format.
5.  The system sends the address to the geocoding service.
6.  The geocoding service returns geographic coordinates.
7.  The system normalizes the location to a canonical location ID.
8.  The system computes the property value estimate.
9.  The system displays the estimated value and range to the user.

## Extensions

-   **4a**: Address format is invalid
    -   4a1: The system displays a validation error message.\
    -   4a2: The user corrects the address and resumes at Step 3.
-   **6a**: Geocoding service fails or returns no match
    -   6a1: The system informs the user that the address could not be
        found.\
    -   6a2: The user may re-enter a different address (resume at Step
        3).\
    -   6a3: The use case ends in Failed End Condition.
-   **8a**: Required data for valuation is partially unavailable
    -   8a1: The system computes a partial estimate using available
        data.\
    -   8a2: The system displays a warning indicating missing data.

## Related Information

-   **Priority**: Must\
-   **Frequency**: High --- expected to be one of the most common entry
    methods.\
-   **Open Issues**:
    -   Selection of geocoding provider.\
    -   Handling of ambiguous or multiple address matches.\
    -   Caching strategy for repeated address lookups.
