# Use Case UC-05: Estimate Property Value Using Location Only

**Goal in Context**: Allow a general user to obtain a property value
estimate using only a location (address, coordinates, or map click)
without providing additional property attributes such as size, bedrooms,
or bathrooms.

**Scope**: Property Value Estimator System

**Level**: User Goal

**Primary Actor**: General User

**Secondary Actors**: Normalization Service; Valuation Engine; Spatial
Database

**Trigger**: The user submits a location input without providing
additional property details.

## Success End Condition

-   The system produces and displays an estimated property value (with
    range) based solely on location-derived features and baseline
    assessment data.
-   The system clearly indicates that the estimate is based on limited
    input.

## Failed End Condition

-   No estimate is produced.
-   The system informs the user that the location could not be
    processed.

## Preconditions

-   The Property Value Estimator system is operational.
-   The provided location input (address, coordinates, or map click) is
    valid.
-   Required baseline assessment data is available for the normalized
    location.
-   The location has been successfully normalized to a canonical
    location ID.

## Main Success Scenario

1.  The user provides a valid location input (address, coordinates, or
    map click).
2.  The system normalizes the input to a canonical location ID.
3.  The system detects that no additional property attributes have been
    provided.
4.  The system retrieves baseline assessment data and location-based
    features associated with the canonical location ID.
5.  The system computes an estimated property value using only available
    location-derived data.
6.  The system computes a low/high estimate range reflecting uncertainty
    due to limited input.
7.  The system displays:
    -   A single estimated value\
    -   A low/high range\
    -   A visible indication that the estimate is based on location-only
        input

## Extensions

-   **2a**: Location normalization fails
    -   2a1: The system informs the user that the location could not be
        processed.\
    -   2a2: The use case ends in Failed End Condition.
-   **4a**: Baseline assessment data is unavailable for the canonical
    location
    -   4a1: The system attempts to compute an estimate using fallback
        spatial averages (e.g., neighbourhood or grid-level averages).\
    -   4a2: The system displays a warning indicating reduced accuracy.\
    -   4a3: The scenario resumes at Step 7.
-   **5a**: Insufficient data to compute even a fallback estimate
    -   5a1: The system informs the user that an estimate cannot be
        generated due to insufficient data.\
    -   5a2: The use case ends in Failed End Condition.

## Related Information

-   **Priority**: Must\
-   **Frequency**: High --- expected default mode when users provide
    minimal input.\
-   **Open Issues**:
    -   Method for calculating uncertainty range with minimal input.\
    -   Clear UX messaging to prevent over-trust in location-only
        estimates.\
    -   Rules for fallback averaging hierarchy (parcel → grid →
        neighbourhood).
