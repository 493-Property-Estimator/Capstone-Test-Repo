# Use Case UC-03: Select Location by Clicking on Map

**Goal in Context**: Allow a general user to click on an interactive map
to select a location so that the system can estimate the property value
for that location quickly and intuitively.\
**Scope**: Property Value Estimator System\
**Level**: User Goal\
**Primary Actor**: General User\
**Secondary Actors**: Map Rendering Service; Spatial Normalization
Service\
**Trigger**: The user clicks on a location within the interactive map
interface.

## Success End Condition

-   The system captures the clicked map coordinates.
-   The location is normalized to a canonical location ID.
-   The system displays an estimated property value (with range) for the
    selected location.

## Failed End Condition

-   No estimate is produced.
-   The system informs the user that the selected location cannot be
    processed.

## Preconditions

-   The Property Value Estimator system is operational.
-   The interactive map interface is loaded and functional.
-   Spatial datasets required for normalization and valuation are
    available.

## Main Success Scenario

1.  The user opens the interactive map interface.
2.  The system displays the map with relevant layers.
3.  The user clicks on a specific point on the map.
4.  The system captures the geographic coordinates of the clicked
    location.
5.  The system verifies that the coordinates fall within the supported
    geographic boundary.
6.  The system converts the coordinates into a canonical location ID.
7.  The system computes the property value estimate using the assessment
    baseline and open-data features.
8.  The system displays the estimated value and range to the user at or
    near the clicked location.

## Extensions

-   **3a**: Click occurs outside the supported geographic boundary
    -   3a1: The system informs the user that the selected location is
        outside the supported area.\
    -   3a2: The use case ends in Failed End Condition.
-   **4a**: Coordinates cannot be resolved due to map or rendering error
    -   4a1: The system displays an error message indicating the
        location could not be determined.\
    -   4a2: The user may click again to retry (resume at Step 3).
-   **7a**: Required valuation data is partially unavailable
    -   7a1: The system computes a partial estimate using available
        data.\
    -   7a2: The system displays a warning indicating missing data.

## Related Information

-   **Priority**: Should\
-   **Frequency**: High --- expected to be a common exploratory
    interaction.\
-   **Open Issues**:
    -   Precision level of click-to-coordinate mapping.\
    -   Handling clicks that fall between parcels.\
    -   Performance implications of rapid repeated clicks.
