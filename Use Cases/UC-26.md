# Use Case 26: Show Missing-Data Warnings in UI

**Goal in Context**: A user is informed when an estimate was generated
with missing or approximated factors (e.g., no crime data, no routing
distance, no work-area scoring) so they do not over-trust the estimate
and can interpret confidence correctly.\
**Scope**: Property Value Estimator UI + Estimate API\
**Level**: User Goal\
**Primary Actor**: General User\
**Secondary Actors**: Estimate API; Valuation Engine\
**Trigger**: User requests an estimate and receives a response that
includes missing-data flags or reduced confidence.

## Success End Condition

-   The UI clearly communicates missing or approximated data and
    presents a confidence/completeness summary without preventing the
    user from using the estimate.

## Failed End Condition

-   The UI fails to show missing-data warnings, causing the user to
    interpret the estimate as fully reliable.

## Preconditions

-   Estimate API response includes metadata describing missing factors
    and completeness.
-   UI has a component to render warnings and confidence indicators.

## Main Success Scenario

1.  **User** selects a property on the map.
2.  **User** requests an estimate (button click or auto-estimate
    trigger).
3.  **System** calls the Estimate API with the property reference.
4.  **Estimate API** returns an estimate response containing:
    -   baseline assessment value,
    -   final estimated price,
    -   factor breakdown,
    -   completeness/confidence score,
    -   missing-data indicators (e.g., missing crime, missing routing).
5.  **UI** parses the response and identifies missing or approximated
    factors.
6.  **UI** displays a confidence indicator (e.g., high/medium/low).
7.  **UI** displays a warning panel listing missing factors (e.g.,
    "Crime data unavailable in this region").
8.  **UI** allows the user to expand details showing how missing factors
    affected the estimate.
9.  **User** reviews warnings and continues exploring or adjusting
    filters.

## Extensions

-   **4a**: Many factors are missing (very low confidence)
    -   **4a1**: UI shows a high-severity warning banner.
    -   **4a2**: UI recommends user widen search radius or select
        another nearby area for comparison.
-   **4b**: Only minor factors missing (small confidence reduction)
    -   **4b1**: UI shows a small non-blocking notice (tooltip or icon).
-   **7a**: Missing factor is due to fallback computation (routing →
    straight-line)
    -   **7a1**: UI displays "Routing unavailable; used straight-line
        approximation."
    -   **7a2**: UI tags the estimate as "Approximate distances used."
-   **7b**: Missing factor is due to dataset coverage gaps
    -   **7b1**: UI displays message explaining "Data not available for
        rural region."
-   **8a**: User dismisses warnings
    -   **8a1**: UI collapses warning panel but keeps a small warning
        icon visible.
    -   **8a2**: User can reopen warnings later.
-   **5a**: Estimate API returns incomplete metadata
    -   **5a1**: UI displays generic warning: "Some data may be
        missing."
    -   **5a2**: System logs the malformed response for debugging.

## Related Information

-   **Priority**: Should
-   **Frequency**: High
-   **Open Issues**:
    -   Define severity thresholds for warnings (what qualifies as low
        confidence).
    -   Decide UI language to avoid confusing users while still being
        transparent.
    -   Decide whether warnings should affect default slider/weight
        behavior.
